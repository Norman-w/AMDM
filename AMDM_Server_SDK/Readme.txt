一个医院有多个付药机
每个付药机提供相关接口.
一个医院还有一个服务器
服务器负责统一获取数据,当需要获取某一个机器的信息的时候,从某一个机器上获取数据.

该项目就是作为服务端的sdk

服务器上的网页通过调用这个sdk中的接口来获取相关的信息

主要包含的信息有:
分仓的信息(api转发)
日志服务器信息(看门狗信息)
文件服务器信息
数据库状态信息等.

服务器的看门狗需要单独记录看门狗服务的安装,启动等日志,如果看门狗连接日志数据库失败,记录的内容应该是直接记录到文件.
其余的关于客户机的日志以及其他服务器相关的日志都记录到数据库

等

该sdk内提供不同的request以及处理request的方法 以Do+request名称的方法
